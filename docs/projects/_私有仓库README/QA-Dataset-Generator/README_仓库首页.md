# 问答数据集生成器

## 项目概述

### 项目目标
开发一个交互式系统，利用大模型（通过 OpenAI API 调用 `qwen-plus` 或其他模型）生成不重复的问题，用户通过图形界面回答问题，大模型对问题和答案进行润色，用户确认后将问答对追加保存到指定的 JSON 文件中，最终生成或扩展一个高质量的问答数据集。

---

## 功能说明

### 1. JSON 文件选择与加载
- **功能描述**：用户启动程序后选择一个 JSON 文件作为基础数据集，后续问答对在此文件上追加。
- **实现方式**：
  - 使用文件选择对话框加载或创建 JSON 文件。
  - 若文件为空或不存在，初始化为空列表 `[]`。

### 2. 问题生成
- **功能描述**：通过大模型 API 生成一个新问题，并确保与历史问题不重复。
- **实现方式**：
  - 用户可在顶部的提示词文本框中输入自定义提示词。
  - 调用 `get_completion` 函数生成问题，避免重复。
  - 用户可在左侧文本框中手动修改生成的问题。

### 3. 答案输入与润色
- **功能描述**：用户输入答案后，通过大模型 API 对问题和答案进行润色。
- **实现方式**：
  - 用户在左侧输入框中输入答案。
  - 点击“提交答案”按钮后，程序会同时润色问题和答案。
  - 润色后的问题和答案会分别显示在对应的文本框中。

### 4. 数据保存
- **功能描述**：将确认后的问答对追加到 JSON 文件中。
- **实现方式**：
  - 用户点击“确认”按钮后，保存当前问题和润色后的答案到 JSON 文件。
  - 数据集内容会同步更新并显示在右侧文本框中。

### 5. 数据集编辑与保存
- **功能描述**：用户可在右侧文本框中查看和编辑当前加载的数据集内容，并保存修改。
- **实现方式**：
  - 右侧文本框显示格式化的 JSON 数据。
  - 用户可手动编辑数据并点击“保存数据集”按钮保存修改。

### 6. 数据集与个人经历合并
- **功能描述**：通过大模型将 JSON 数据集与个人经历合并，生成新的个人经历。
- **实现方式**：
  - 点击“合并数据集和个人经历”按钮，调用大模型生成合并后的个人经历。
  - 合并结果显示在润色结果框中。

### 7. 替换个人经历
- **功能描述**：将合并后的个人经历保存到 `config.py` 文件中，替换原有的 `PERSONAL_EXPERIENCE`。
- **实现方式**：
  - 点击“确认保存并更新个人经历”按钮，将合并后的个人经历写入 `config.py` 文件。

### 8. 程序关闭自动保存
- **功能描述**：程序关闭时自动保存当前数据集内容。
- **实现方式**：
  - 在 `closeEvent` 方法中实现自动保存逻辑。

## 系统架构

### 界面布局
1. **顶部**：
   - 提示词输入框：用户可输入自定义提示词控制问题生成。
   - 模型选择下拉框：用户可选择不同的大模型（如 `qwen-plus` 或 `qwen-14b-chat`）。
   - 总结按钮：总结问答对和个人经历。
   - 替换按钮：替换个人经历。
2. **左侧**：
   - 问题显示框：显示生成的问题，支持手动修改。
   - 答案输入框：用户输入答案。
   - 润色结果框：显示润色后的答案，支持手动修改。
   - 按钮：换一个问题、提交答案、确认。
3. **右侧**：
   - 数据集显示框：显示当前加载的数据集内容，支持手动编辑。
   - 保存按钮：保存编辑后的数据集。
   - 合并按钮：合并数据集和个人经历。
   - 确认保存按钮：保存并更新个人经历。

## 技术选型

- **前端**：PyQt5（提供跨平台的图形界面支持）。
- **大模型 API**：OpenAI 兼容接口，调用阿里云 DashScope 的 `qwen-plus` 或其他模型。
- **编程语言**：Python 3.8+。
- **依赖库**：
  - `PyQt5`：用于图形界面开发。
  - `openai`：用于调用大模型 API。
  - `json`：用于数据集的加载和保存。
